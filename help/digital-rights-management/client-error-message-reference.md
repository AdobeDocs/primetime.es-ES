# Referencia de mensajes de error del cliente DRM {#client-error-message-reference}

Los errores del cliente DRM son un subconjunto de los errores del cliente TVSDK, con códigos de errores relacionados con DRM que van desde 3300 a 3399.

## Errores del cliente DRM {#drm-client-errors}

| Código de error | Mnemotécnico | Remedy |
|---|---|---|
| 3300 | Cupón no válido | Qué debe hacer el software del distribuidor: <ul><li> Si está utilizando Google Chrome y está en modo Incognito, y la versión de Flash Player es inferior a 11.6, este error podría producirse. Recomendamos que el reproductor compruebe el número de versión del navegador y que recomiende al usuario que salga del modo Incognito.</li><li> Vuelva a solicitar la licencia. Si la solicitud se realiza correctamente, no es necesario que se registre o escale. Si la solicitud no se realiza correctamente, registre el contenido que provocó el error. `subErrorId` contiene un error de línea si está presente.</li></ul> Qué debe hacer el distribuidor: <ul><li> Si los reintentos no tienen éxito en configuraciones distintas de Chrome con Flash menor que la versión 11.6, puede que se haya producido un error en el empaquetado.</li><li>Compruebe si el problema es específico de cierto contenido y vuelva a empaquetarlo.</li></ul> |
| 3301 | Error de autenticación | El servidor no pudo autenticar ni autorizar el cliente.<ul><li>El software del distribuidor debe tomar todas las medidas necesarias para restablecer las credenciales del usuario o guiarlo para obtener acceso al contenido.</li><li>El distribuidor debe confirmar que su mecanismo de autorización y autenticación funciona correctamente.
