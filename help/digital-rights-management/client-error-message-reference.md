# Referencia de mensaje de error del cliente DRM {#client-error-message-reference}

Los errores del cliente DRM son un subconjunto de los errores del lado del cliente de TVSDK, con códigos de errores relacionados con DRM que van desde 3300 a 3399.

## Errores de cliente DRM {#drm-client-errors}

| Código de error | Mnemotécnico | Solución |
|---|---|---|
| 3300 | Cupón no válido | Qué debe hacer el software del distribuidor: <ul><li> Si está utilizando Google Chrome, y está en modo Incognito, y su versión de Flash Player es inferior a la 11.6, este error podría ocurrir. Se recomienda que el reproductor compruebe el número de versión del navegador y que se aconseje al usuario que salga del modo Incognito.</li><li> Vuelva a solicitar la licencia. Si la solicitud se realiza correctamente, no es necesario que registre ni escale. Si la solicitud no se realiza correctamente, registre el contenido que provocó el error. `subErrorId` contiene un error de línea si está presente.</li></ul> Qué debe hacer el distribuidor: <ul><li> Si los reintentos no tienen éxito en configuraciones que no sean Chrome con un Flash inferior a la versión 11.6, es posible que se haya producido un error en el paquete.</li><li>Compruebe si el problema es específico de cierto contenido y vuelva a empaquetar.</li></ul> |
| 3301 | Error de autenticación | El servidor no pudo autenticar ni autorizar al cliente.<ul><li>El software del distribuidor debe realizar cualquier acción necesaria para restablecer las credenciales del usuario o guiarlo para que adquiera acceso al contenido.</li><li>El distribuidor debe confirmar que su mecanismo de autorización y autenticación funciona correctamente. Si los distribuidores no tienen previsto utilizar las funciones de autenticación o autorización, deben comprobar si la directiva del contenido infractor requiere autenticación y ver Diagnóstico de discrepancias de directivas/licencias.</li></ul> Para obtener más información sobre este código de error, consulte [DRM Error 301 causes and resolution](https://forums.adobe.com/thread/1277149). |
| 3302 | RequireSSL | Para Primetime DRM 4.0 y posteriores, este error se lanza a iOS cuando la URL de clave remota no utiliza HTTPS como esquema. Se requiere HTTPS. <ul><li>Si el distribuidor utiliza una versión anterior a Primetime DRM versión 4, o al menos versión 4, pero la plataforma no es iOS, el software del distribuidor debe registrar el error. Este error solo se produce en iOS.</li><li>Si el software del distribuidor es al menos Primetime DRM versión 4, y la plataforma es iOS, los distribuidores deben cambiar la URL del servidor de claves remotas que utilizan a HTTPS. Si solo utilizaran HTTP, es posible que los distribuidores tengan que configurar un servidor HTTPS. De lo contrario, los distribuidores deben enviar la información registrada a Adobe y escalar el problema.</li></ul> |
| 3303 | ContentExpired | El contenido que está viendo ha caducado según las reglas establecidas por el proveedor de contenido.  `subErrorId` contiene un error específico del cliente o un error de línea. <ul><li>El software del distribuidor debe intentar volver a adquirir una licencia del servidor una vez para determinar si hay una nueva licencia no caducada disponible. La herramienta de línea de comandos Java es: <ul><li>java -jar libs/AdobePolicyManager.jar</li><li> detail demo.pol</li><li>El distribuidor debe confirmar si las fechas de caducidad de la licencia están configuradas según lo previsto.</li></ul> Para obtener más información sobre este código de error, consulte [3303 (Contenido caducado) con AMS/FMS mediante un flujo en directo?](https://forums.adobe.com/thread/1300813) |
| 3304 | Error de autorización | El usuario actual no está autorizado para ver contenido. Inicie sesión como un usuario diferente.<br>Para obtener más información sobre este código de error, consulte Código de error 3301. |
| 3305 | ServerConnectionFailed | Se agotó el tiempo de espera de la conexión a los servidores de dominio o licencia, ya sea debido a un retraso de la red o a que el cliente no tiene conexión.
